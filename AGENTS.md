# AGENTS.md — enclave-sign-api

Supabase edge functions and migrations for **Enclave Sign**.

## Layout

```text
supabase/functions/
  exchange-account-token/
  sign-open-envelope/
  sign-complete-recipient/
  sign-send-invite/
  sign-download-completed/
  _shared/sign/           Handler helpers (PDF, decrypt, session)
  _vendor/                Generated by npm run vendor (gitignored)
  deno.json               Import map for vendored SDK + noble pins
scripts/vendor-sign-sdk.mjs
supabase/migrations/
```

## Commands

```bash
npm run vendor
npm run deploy
```

## Vendor sources (sibling repos)

| Source | Vendored to |
|--------|-------------|
| `Enclave-Sign/enclave-sign-sdk/dist` | `_vendor/sign-sdk/` |
| `Enclave-Inc/enclave-pqc-primitives/dist` | `_vendor/pqc-primitives/` |
| `registry/ENCLAVE_PQ_SUITE_v1.json` | `_vendor/ENCLAVE_PQ_SUITE_v1.json` |

## Build before vendor

```bash
cd ../../Enclave-Inc/enclave-pqc-primitives && npm run build
cd ../../Enclave-Sign/enclave-sign-sdk && npm run build
cd ../../Enclave-Encrypt/enclave-sign-api && npm run vendor
```

## Rules

1. Import `@enclave/sign-sdk/*` via `deno.json` — not sibling repo paths at runtime.
2. Do not import `@enclave/pqc-primitives` directly unless adding a new SDK entry point.
3. Run `npm run vendor` after sign-sdk or primitives changes, before deploy.

## Dependency chain

```text
handlers → @enclave/sign-sdk → @enclave/pqc-primitives
```
